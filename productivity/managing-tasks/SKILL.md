# Managing Tasks

## Scope
Operating a single trusted system for all commitments: capture, clarify, organize,
review, and do. Covers the whole lifecycle of a task from inbox to done — not
scheduling a specific day or splitting one big task into steps.

## Purpose
Get every commitment out of your head into one system you trust, each phrased as a
doable next action — so attention goes to doing the work, not remembering it.

## When to use this skill
- "Organize my tasks / set up a to-do system / I keep forgetting things."
- Commitments are scattered across apps, notes, email, and memory.
- The existing list is stale, vague, or ignored because it isn't trusted.

## When NOT to use this skill
- Breaking one large task into sub-steps → [breaking-down-tasks](../breaking-down-tasks/SKILL.md).
- Scheduling today's chosen tasks onto the calendar → [planning-your-day](../planning-your-day/SKILL.md).
- Team/project tracking with owners and status → [planning-projects](../../business/planning-projects/SKILL.md).

## Inputs
- Everything currently owed: emails, notes, sticky pads, mental loops, half-lists.
- Existing tools (whatever they are) and the contexts work happens in (@computer, @calls, @errands).
- The active projects each loose task belongs to.

## Outputs
- One trusted system with an inbox, a next-actions list, projects, and someday/maybe.
- Every actionable item phrased as a concrete verb-first next action.
- A recurring review cadence (weekly) that keeps the system current.

## Workflow
```
Progress:
- [ ] 1. Capture everything into one inbox
- [ ] 2. Clarify each item: actionable? what's the next action?
- [ ] 3. Decide: do (<2 min), delegate, defer, or drop
- [ ] 4. Organize into next-actions, projects, someday/maybe by context
- [ ] 5. Do from the right list for your context and energy
- [ ] 6. Review weekly to keep the system trusted
```

**Step 1 — Capture.** Empty every source into one inbox — head, notes, email flags.
Capture is not organizing; just get it all in one place first.

**Step 2 — Clarify.** For each item ask: is it actionable? If yes, name the single
concrete next physical action ("email Sam the draft"), not a vague topic ("proposal").

**Step 3 — Decide.** If it takes under two minutes, do it now. Otherwise delegate,
defer to a list, or drop it. Multi-step outcomes become projects with one next action.

**Step 4 — Organize.** File next actions by context so the list matches where you
are. Keep projects (outcomes) separate from actions; park not-now items in someday/maybe.

**Step 5 — Do.** Work from the next-actions list filtered by context, time, and
energy — no re-deciding mid-flow. Schedule the chosen items via [planning-your-day](../planning-your-day/SKILL.md).

**Step 6 — Review.** Weekly, empty the inbox, mark done, add next actions to stalled
projects, and prune. The review is what makes the system trustworthy.

## Principles
1. **One system, not five** — competing lists mean no list is trusted.
2. **Capture everything** — the mind is for having ideas, not holding them.
3. **Every item is a next action** — verb first, concrete, doable.
4. **Projects ≠ actions** — an outcome always has one visible next step.
5. **Context over priority for retrieval** — surface what's doable where you are.
6. **The weekly review is non-negotiable** — without it, trust decays.

## Decision framework
- **Under two minutes?** Do it now, don't file it.
- **Not actionable but might matter?** Someday/maybe or reference, not next actions.
- **Multi-step?** Make it a project; write the single next action.
- **Action stated vaguely?** Rewrite verb-first until it's physically doable.
- **List feels stale?** You skipped reviews — run one now.

## Common mistakes
- **Multiple competing task lists** across apps so none is trusted.
- **Vague, non-actionable entries** ("website", "mom") that stall on sight.
- **Projects listed as if they were single tasks** with no next action.
- **Capturing but never clarifying** — an inbox that becomes a swamp.
- **Skipping the weekly review** until the system rots and gets abandoned.
- **Over-tooling** — endless app-hopping instead of doing the work.

## Validation checklist
- [ ] Exactly one inbox and one next-actions system in use.
- [ ] Every actionable item is phrased verb-first and concrete.
- [ ] Projects are tracked separately, each with a visible next action.
- [ ] Non-actionable items are in someday/maybe or reference, not mixed in.
- [ ] Contexts/tags let the right list surface for the moment.
- [ ] A weekly review is scheduled and actually happening.

## Edge cases
- **Analog vs digital:** medium doesn't matter; "one trusted system" does.
- **Shared/team work:** track your commitments here; use project tools for team status.
- **Waiting-on items:** a dedicated "waiting for" list so delegated work isn't lost.
- **Overwhelming backlog:** timebox a first pass; park the rest in someday/maybe.

## Examples
**Input:** "My tasks are everywhere and I keep dropping things — help me set up a system."
**Output:** Captured 40+ items from inbox, notes, and memory into one inbox. Clarified
each: did the seven under-two-minute items on the spot; rewrote "proposal" as "draft
proposal outline (@computer)"; turned "kitchen reno" into a project with next action
"call contractor for quote (@calls)". Organized by context, moved 12 not-now items to
someday/maybe. Set a Friday 30-min weekly review. Now one trusted list, all actionable.

## Automation opportunities
- Quick-capture shortcut (widget/hotkey) feeding the single inbox from anywhere.
- Email-to-task and flag rules so nothing lives only in the inbox.
- Recurring weekly-review reminder with a saved checklist.
- Saved context filters (@computer, @calls, @errands) for one-tap retrieval.
