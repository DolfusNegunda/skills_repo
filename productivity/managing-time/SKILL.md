# Managing Time

## Scope
Defending attention and executing focused work once the priorities are chosen:
deep-work blocks, batching similar tasks, single-tasking, timeboxing/Pomodoro, and
saying no to low-value demands. About *how* to work, not *what* to work on.

## Purpose
Convert scheduled hours into actual finished work by protecting sustained attention —
so the important tasks get deep, uninterrupted effort instead of being nibbled away.

## When to use this skill
- "I can't focus / always busy but nothing ships / stop context-switching."
- Deep work keeps getting eaten by meetings, pings, and interruptions.
- Time leaks into shallow tasks and reactive demands with nothing to show.

## When NOT to use this skill
- Deciding *what* to work on → [managing-tasks](../managing-tasks/SKILL.md) or [prioritizing-options](../../reasoning/prioritizing-options/SKILL.md).
- Laying out the day's schedule → [planning-your-day](../planning-your-day/SKILL.md).
- Splitting a big task into steps → [breaking-down-tasks](../breaking-down-tasks/SKILL.md).

## Inputs
- The chosen priorities and their calendar blocks (from [planning-your-day](../planning-your-day/SKILL.md)).
- Current interruption sources: notifications, meetings, drop-ins, self-checks.
- Your attention pattern: how long you can focus and when it peaks.

## Outputs
- Protected deep-work blocks with interruptions actively suppressed.
- Batched shallow work and a single-tasking discipline for the focus window.
- A default "no" or defer for low-value demands, freeing hours for the priorities.

## Workflow
```
Progress:
- [ ] 1. Identify the top interruption sources and time leaks
- [ ] 2. Reserve deep-work blocks in peak-energy hours
- [ ] 3. Eliminate distraction for each block (notifications, tabs, status)
- [ ] 4. Single-task one block; use timeboxing/Pomodoro to sustain it
- [ ] 5. Batch shallow work into dedicated windows
- [ ] 6. Decline or defer low-value demands to protect the blocks
```

**Step 1 — Diagnose.** Notice where attention actually goes — notifications, meetings,
drop-ins, self-interruption. Name the top few leaks before fixing anything.

**Step 2 — Reserve.** Put deep-work blocks in peak-energy hours and treat them as
fixed appointments. See [planning-your-day](../planning-your-day/SKILL.md) for placement.

**Step 3 — Eliminate distraction.** Before each block: silence notifications, close
unrelated tabs, set status to busy, phone out of reach. Remove friction to starting.

**Step 4 — Single-task.** One task per block. Use a timebox or Pomodoro (e.g. 25/5)
to sustain and rest attention. When a stray thought lands, note it and return.

**Step 5 — Batch.** Group email, messages, and admin into 1–2 windows instead of
constant switching — each context switch costs re-focus time.

**Step 6 — Decline.** Say no or defer to low-value meetings and requests; suggest
async or a shorter slot. Protecting the block is the point.

## Principles
1. **Attention is the scarce resource,** not time — protect it first.
2. **Single-task in focus blocks** — multitasking is serial switching with a tax.
3. **Context switches are expensive** — batch to pay the cost once.
4. **Deep work is scheduled and defended,** never hoped for.
5. **A calm "no" protects the yes** you already made to your priorities.
6. **Match block length to your real attention span,** then grow it.

## Decision framework
- **Interrupted mid-block?** Capture the thought, don't act on it, resume.
- **Meeting invite in a focus block?** Decline, propose async, or offer a shallow slot.
- **Can't sustain a long block?** Shorten it (Pomodoro) rather than abandon it.
- **Urgent vs important clash?** Batch the urgent-shallow; guard the important-deep.
- **Always reactive?** Reserve one non-negotiable morning block before the inbox opens.

## Common mistakes
- **Multitasking** — believing parallel work is faster than sequential focus.
- **Back-to-back meetings** leaving zero recovery or focus time.
- **Notifications left on** during deep work, guaranteeing fragmentation.
- **Saying yes by default** to every request and meeting.
- **Checking email/chat constantly** instead of in batches.
- **Blocks too long for actual stamina,** so they collapse and get abandoned.

## Validation checklist
- [ ] At least one distraction-free deep-work block runs daily.
- [ ] Notifications are silenced during focus blocks.
- [ ] Shallow work is batched into defined windows, not continuous.
- [ ] One task per focus block (no multitasking).
- [ ] Low-value meetings/requests are declined, shortened, or made async.
- [ ] Block length matches sustainable attention (adjusted from experience).

## Edge cases
- **Open-plan/noisy space:** signal unavailability (headphones, status, sign); use short blocks.
- **On-call/support role:** carve one protected block; batch the rest around triage.
- **Cross-time-zone teams:** batch async replies; guard local peak hours for deep work.
- **ADHD/variable focus:** shorter timeboxes, external cues, frequent short breaks.

## Examples
**Input:** "I'm busy all day but never finish real work — help me focus."
**Output:** Diagnosed the leaks: notifications every few minutes and five back-to-back
meetings. Reserved a 9–11 deep-work block before the inbox; silenced notifications and
set status to busy; single-tasked with 25/5 Pomodoros. Batched email into 12:00 and
4:30 windows. Declined two optional meetings and made a third async. Result: the
priority shipped by 11, and reactive work fit the afternoon.

## Automation opportunities
- Focus/Do-Not-Disturb mode that auto-enables during calendar focus blocks.
- Notification schedules that mute chat outside batch windows.
- Auto-decline or "tentative" rules for meetings overlapping protected blocks.
- A Pomodoro timer wired to a start-of-block shortcut.
