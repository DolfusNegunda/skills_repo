# Citing Sources

## Scope
Correct, consistent attribution: selecting a citation style, formatting in-text
citations and reference lists, and attributing ideas/quotes/data to avoid plagiarism.
Applies to any cited work. Judging source quality is
[assessing-source-credibility](../assessing-source-credibility/SKILL.md).

## Purpose
Give credit accurately and make every claim traceable — using the correct style
consistently so readers can find each source and no work is passed off unattributed.

## When to use this skill
- "Cite this / format references or a bibliography / which citation style?"
- "Add citations / check attribution / avoid plagiarism."
- Preparing references for a paper, report, or article.

## When NOT to use this skill
- Judging whether a source is trustworthy → [assessing-source-credibility](../assessing-source-credibility/SKILL.md).
- Verifying a claim is true → [verifying-facts](../verifying-facts/SKILL.md).
- Gathering the sources → [collecting-evidence](../collecting-evidence/SKILL.md).

## Inputs
- The sources (with full metadata) and the required/appropriate citation style.
- The document type and field (drives style choice).

## Outputs
- Correctly formatted in-text citations and a complete, consistent reference list in
  the chosen style, with every borrowed idea/quote/data point attributed.

## Workflow
```
Progress:
- [ ] 1. Choose the style (given, or by field convention)
- [ ] 2. Collect complete metadata per source
- [ ] 3. Format in-text citations consistently
- [ ] 4. Build the reference list/bibliography in style
- [ ] 5. Attribute every borrowed idea, quote, and data point
- [ ] 6. Check consistency and completeness
```

**Step 1 — style by field:** APA (social sciences), MLA (humanities), Chicago
(history/publishing; notes or author-date), IEEE (engineering), Vancouver
(medicine). Use the required one; if unspecified, follow the field convention — and
then be **consistent**. **Step 2 — complete metadata:** author, title, year,
publisher/journal, volume/issue/pages, DOI/URL, access date for web. Missing metadata
is the top citation defect. **Step 5 — attribute everything borrowed:** direct quotes
(quotation marks + citation), paraphrases (citation), and data/ideas alike.
Paraphrasing without citation is still plagiarism. **Step 6 — consistency:** one
style, applied uniformly; in-text citations and the reference list must match.

## Principles
1. **One style, applied consistently.**
2. **Complete metadata** — enough for a reader to find the source.
3. **Attribute all borrowed content** — quotes, paraphrases, data, ideas.
4. **In-text and reference list must correspond** exactly.
5. **Accuracy over approximation** — a wrong citation is worse than none.
6. **Traceability** — every claim leads to a findable source.

## Decision framework
- **Style specified?** Use it exactly. **Not specified?** Field convention, then consistency.
- **Direct quote?** Quotation marks (or block) + citation + page/locator.
- **Paraphrase/idea?** Citation, no quotation marks.
- **No author/date?** Use the style's rules for missing elements (n.d., organization as author).
- **Web source?** Include access date if the style requires and content may change.

## Common mistakes
- **Mixing styles** within one document.
- **Incomplete references** — missing year, pages, DOI.
- **Paraphrasing without citing** — still plagiarism.
- **In-text citations with no matching reference entry** (or vice versa).
- **Fabricated/guessed citation details** — never invent metadata.
- **Wrong locator** for a quote.

## Validation checklist
- [ ] A single appropriate style chosen and applied throughout.
- [ ] Every source has complete metadata.
- [ ] In-text citations formatted correctly and consistently.
- [ ] Reference list complete and in style.
- [ ] All quotes, paraphrases, data, and ideas attributed.
- [ ] In-text citations and reference entries correspond 1:1.
- [ ] No fabricated or approximated citation details.

## Edge cases
- **Missing metadata:** use the style's conventions (n.d., no author); don't invent.
- **Secondary citation (as cited in):** cite per the style; prefer finding the original.
- **Non-standard sources (datasets, software, AI tools):** follow the style's specific rules.
- **Self-citation/prior work:** still cite to avoid self-plagiarism.

## Examples
**Input:** "Add APA citations for these three studies in my report."
**Output:** In-text author-date citations (e.g. "(Chen & Patel, 2024)") placed at each
borrowed claim, and a matching APA reference list with full metadata (authors, year,
title, journal, volume(issue), pages, DOI) — consistent throughout, every claim
traceable, nothing fabricated.

## Automation opportunities
- Maintain source metadata in a reference manager; generate citations in any style.
- Run a consistency check that every in-text citation has a reference entry and vice versa.
