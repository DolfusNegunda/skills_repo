A skills library for everyday office work

Nine reusable, house-style skills that teach Claude to do the document and communication work every team repeats — meeting notes, emails, status reports, branded deliverables, document summaries, and more. Each is a self-contained folder Claude loads only when it's relevant.

Note (2026-07): this page showcases the original nine office skills. The library now spans 151 skills across ten areas (office, review, business, research, software-/data-/AI-engineering, reasoning, productivity, plus the meta-skill). See skills-index.md for the full current set.

9 skills works in Claude Code, the API & claude.ai no external services · no MCP required

How the pieces fit together

The skills are designed to compose. The core pipeline turns raw updates into a branded, shippable deliverable — content skills produce the substance, and one rendering skill applies the standard look and the right client logo.

05

Using these skills

Every skill is a folder with a SKILL.md. Claude loads one when your request matches its description, or you can invoke it directly.

In Claude Code

Place or symlink a skill folder under .claude/skills/, then type /skill-name — or just describe the task and Claude auto-loads the match.

ln -s .../summarizing-meeting-notes \
  ~/.claude/skills/summarizing-meeting-notes

Share with the team

Commit a .claude/skills/ folder into a shared project repo — everyone gets the same skills on checkout, and the brand/templates stay a single source of truth.

Extend or add one

Use /skill-builder to author new skills to the same standard: strong description, lean body, a concrete example, and a validation step.

Design principle across the library: document skills are thin house-style layers over Claude's built-in document generation — they carry your templates, tone, and checklists, and never reimplement file-format internals or depend on an external MCP server.
