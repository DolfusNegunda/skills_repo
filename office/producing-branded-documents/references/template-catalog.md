## weekly-project-update (Word)

- **File:** `assets/templates/weekly-project-update.docx`
- **Use for:** weekly status sent to a client or internal stakeholders.
- **Generated by:** `scripts/make_template.py`
- **Filled by:** `scripts/fill_docx.py --data <data.json> --out <file.docx>`

**Data fields** (JSON, see `examples/sample_update.json`):

| Field | Type | Notes |
|---|---|---|
| `company_name` | string | Your company (footer/header) |
| `client_name` | string | Client display name (header) |
| `client_key` | string | Logo lookup key → `logos/clients/<key>.png` |
| `project_name` | string | |
| `week` | string/number | |
| `report_date` | string | ISO date recommended |
| `prepared_by` | string | |
| `overall_status` | string | `Green`/`Amber`/`Red` or `On track`/`At risk`/`Off track` — auto-colored |
| `summary` | string | One short paragraph |
| `accomplishments` | list[string] | Bulleted |
| `in_progress` | list[string] | Bulleted |
| `blockers` | list[string] | Bulleted |
| `upcoming` | list[string] | Bulleted |
| `notes` | string | Free text |

## Adding a new template

1. Add a `make_<name>.py` (or extend `make_template.py`) that builds the layout
   with the header/footer/logo frame and `{{ jinja }}` body placeholders, using
   the colors/fonts from `assets/brand/brand-spec.md`.
2. Save to `assets/templates/<name>.docx`.
3. Document its fields here.
4. Fill with `fill_docx.py --template assets/templates/<name>.docx ...`.

Keep the header/footer/logo frame identical across templates so every deliverable
type shares one look.

## Adding a new client

Drop `assets/logos/clients/<client-key>.png` (transparent PNG, ≥300px tall).
No template or code changes needed. Set `client_key` in the data to match.
